## Feature: 支持全量快照与增量备份

### Feature 价值论证

**用户痛点**：云端侧集群承载 TB 级时序数据，7x24 高写入场景下任何数据丢失都意味着传感器历史不可重建。备份窗口不能显著影响写入吞吐。合规审计要求备份数据加密存储、操作可追溯。

**现有方案不足**：文件系统级快照无法保证数据一致性（热备份场景下文件处于写入中间状态），停机拷贝对 7x24 场景不可接受。文件系统级快照产出的备份文件明文存储，不满足合规审计对数据静态加密的要求。

**本 Feature 价值**：提供数据库层面的一致性快照 API 和增量备份 API，不停写即可获取一致性数据副本；备份文件落盘即加密，操作全程可审计。**本域只覆盖单集群侧备份恢复，跨边云归档由[边云协同域](../requirements/边云协同.md)覆盖。**

**成功指标**：
- 全量快照期间写入吞吐下降 <= 15%
- 增量备份数据量相比全量备份减少 >= 80%（典型工业传感数据，1h 写入变更量 < 全量 20%）
- RPO=1h 策略下连续 72h 备份窗口达标率 >= 99.9%

### Scenario 清单

| # | 场景 | 验收标准 | 可测试性 |
|---|------|---------|---------|
| 1 | 备份系统在集群持续写入期间触发全量快照 | 快照数据通过一致性校验，快照期间写入吞吐下降 <= 15% | 自动化 |
| 2 | 备份系统基于上一次全量快照执行增量备份（典型工业传感数据，1h 写入变更量 < 全量 20%） | 增量备份数据量 <= 全量快照的 20% | 自动化 |
| 3 | 备份系统按 RPO=1h 策略连续执行多轮增量备份形成完整备份链 | 连续 72h 无断链，任意时间点可追溯到完整恢复路径 | 自动化 |
| 4 | 全量快照执行过程中一个数据节点发生故障重启 | 快照操作返回错误或自动重试成功，不产生不完整快照残留 | 自动化 |
| 5 | 增量备份链中某一环缺失（文件丢失或存储空间不足导致写入失败） | 系统检测到链断裂并返回错误码及原因，已有备份链不受影响，可据此触发新全量快照 | 自动化 |
| 6 | 备份系统对已完成的快照文件执行完整性校验 | 校验 API 返回每个数据块的 checksum 比对结果 | 自动化 |
| 7 | 备份系统完成快照后，安全审计员验证备份文件已加密存储并查询备份操作审计日志 | 备份文件使用 AES-256 加密，审计日志包含操作者、操作类型、时间戳、结果且不可篡改 | 自动化 |
| 8 | 备份系统向远程存储写入快照过程中发生网络分区 | 已传输部分保留在本地缓存，网络恢复后从断点续传，不产生不完整快照残留 | 自动化 |
| 9 | 两个备份任务同时请求对同一数据集创建快照 | 第二个请求被拒绝或排队，已执行快照数据不受干扰 | 自动化 |
| 10 | 数据库升级后读取升级前版本创建的备份链 | 升级前备份链可完整读取，增量备份链可继续追加 | 自动化 |
| 11 | 备份系统按配置的保留期策略自动清理过期备份文件 | 保留期满的备份文件及其关联增量链被删除，清理操作记录审计日志，清理后存储空间下降 | 自动化 |
| 12 | 平台运维工程师调整备份保留期策略（延长或缩短）后，已有备份按新策略重新评估 | 保留期延长时已有备份继续保留；保留期缩短时超期备份在下次清理周期被删除，调整操作记录审计日志 | 自动化 |

## Feature: 支持从备份恢复数据（含 PITR）

### Feature 价值论证

**用户痛点**：灾难性故障（多节点磁盘损坏、误操作删除数据）时需要恢复到故障前精确时间点。恢复粒度越粗，丢失的数据窗口越大。合规审计要求恢复操作有权限控制和操作记录。

**现有方案不足**：仅支持恢复到最近全量快照意味着丢失快照后所有增量数据。无 PITR 能力时误操作后中间正常写入也一并丢失。缺乏恢复操作权限控制意味着任何有数据库访问权限的用户都可触发恢复，存在误操作风险。

**本 Feature 价值**：支持全量快照 + 增量备份 + WAL 重放实现任意时间点恢复（PITR），将数据丢失窗口从"小时级"缩小到"秒级"；恢复操作需授权角色触发且全程可审计。**本域只覆盖单集群侧备份恢复，跨边云归档由[边云协同域](../requirements/边云协同.md)覆盖。**

**成功指标**：
- 实际 RPO <= 1h（备份策略为每小时全量+增量链，恢复后丢失数据窗口不超过最近一次成功备份的时间间隔）
- PITR 恢复精度 <= 1 秒（WAL 保留期内）
- 1TB 数据量全量恢复 RTO <= 30 分钟（3 节点集群、NVMe SSD、本地磁盘）

### Scenario 清单

| # | 场景 | 验收标准 | 可测试性 |
|---|------|---------|---------|
| 1 | 备份系统使用全量快照 + 增量备份恢复到最近备份完成时间点 | 恢复后数据与备份时间点源数据逐条一致 | 自动化 |
| 2 | 备份系统指定任意历史时间戳执行 PITR（假设：3 节点集群、NVMe SSD、WAL 保留期 >= 7 天） | 恢复后数据精度 <= 1s，不包含该时间戳后的写入 | 自动化 |
| 3 | 平台运维工程师在恢复过程中查询进度和预估剩余时间 | 进度 API 返回百分比及预估剩余时间，误差 <= 20% | 自动化 |
| 4 | 恢复过程中目标节点发生磁盘 IO 错误 | 恢复暂停并返回可重试错误码，重试后从断点续传 | 自动化 |
| 5 | 备份系统使用的快照文件存在部分数据块损坏 | 恢复前校验检测到损坏并报告范围，拒绝执行不完整恢复 | 自动化 |
| 6 | PITR 请求的时间点超出可恢复范围（备份覆盖范围外或 WAL 已清理） | 返回明确错误，包含可恢复的最早时间点及不可恢复原因 | 自动化 |
| 7 | 恢复读取远程备份存储期间发生网络分区 | 恢复暂停并返回网络错误，网络恢复后可从断点续传 | 自动化 |
| 8 | 恢复操作执行期间另一操作者尝试对同一数据库触发写入 | 写入请求被拒绝并返回恢复中状态提示，恢复完成后写入正常 | 自动化 |
| 9 | 平台运维工程师在灾后从零重建集群后从备份恢复全部数据（同版本空集群，已安装兼容版本二进制） | 初始化后成功导入全量快照和增量备份链，恢复后 schema 一致、索引可用、元数据完整、数据逐条一致 | 自动化 |
| 10 | 数据库升级后恢复升级前版本创建的备份 | 旧版本备份可正确恢复，恢复后数据完整且可读 | 自动化 |
| 11 | PITR 恢复期间用户发起历史数据查询 | 恢复前已存在的历史数据查询正常返回，查询结果不包含恢复中数据；恢复完成后新数据可查询 | 自动化 |
| 12 | 备份系统将 72h 增量备份链合并为合成全量快照 | 合成全量快照数据与逐层应用增量链结果逐条一致，合成后恢复 RTO 不随链长度增长 | 自动化 |

## 本域 Feature 依赖关系

| 依赖项 | 依赖来源 | 依赖强度 | 说明 |
|--------|----------|----------|------|
| WAL 持久化保障 | 数据写入域 | 强 | 增量备份依赖 WAL 识别变更数据，PITR 依赖 WAL 重放实现秒级精度恢复 |
| 集群节点健康状态 | 集群管理域 | 弱 | 备份期间节点故障检测与自动切换由集群管理域覆盖，本域仅需处理备份任务自身的容错 |
| RBAC 权限体系 | 安全合规域 | 强 | 恢复操作权限控制依赖 RBAC 授权角色判定 |
| 远程存储接入 | 边云协同域 | 弱 | 备份文件远程归档由边云协同域覆盖，本域仅定义本地备份链的恢复能力 |

## 本域验收标准

| 验收项 | 标准描述 | 可测试性等级 |
|--------|----------|--------------|
| 备份一致性 | 全量快照恢复后数据与源数据逐条一致（含时序顺序和值精度） | 自动化 |
| 增量效率 | 典型工业传感数据 1h 增量备份 <= 全量 20% | 自动化 |
| RPO 达标 | RPO=1h 策略下连续 72h 备份窗口达标率 >= 99.9% | 自动化 |
| PITR 精度 | 恢复到指定时间戳误差 <= 1s（WAL 保留期内） | 自动化 |
| 恢复 RTO | 1TB 数据恢复 <= 30 分钟（3 节点集群、NVMe SSD） | 自动化 |
| 备份加密 | 备份文件落盘使用 AES-256 加密 | 自动化 |
| 操作审计 | 备份/恢复操作审计日志不可篡改且可按时间范围查询 | 自动化 |
| 恢复权限 | 仅 RBAC 授权角色可触发恢复操作 | 自动化 |
| 备份存储可观测性 | 备份系统暴露当前备份链总大小、预估下一周期增量大小、存储空间使用率指标，空间不足前 >= 24h 触发告警 | 自动化 |
