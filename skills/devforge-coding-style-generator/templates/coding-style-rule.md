# coding-style 规则条目模板

本模板定义 `coding-style-<lang>.md` 中**单条规则**的强制格式。生成规范的 agent 必须为每条规则严格遵循本模板，禁止自由发挥为段落、表格或其他结构。

## 规则章节结构（从上到下）

规则条目支持两种层级结构，根据规则数量和领域密度选择。

### 平铺结构（规则 ≤ 10 条或无需分组）

````markdown
### {编号}. {规则标题}

**{规则陈述}**

- 适用场景：{什么情况下这条规则生效}
- 原因：{为什么这样写 / 不这样写的错因}

> **注意**：{可选补充说明}
> **例外**：{可选补充说明}

**正确示例**（{文件路径}）：
```{lang}
{来自该仓库真实代码的短片段}
```

**错误示例**（{文件路径}，错误写法）[仅禁止规则必填]：
```{lang}
{该仓库中真实存在的反例片段}
```

**推荐替代写法** [仅禁止规则必填]：
```{lang}
{修正后的写法}
```
````

### 分组结构（规则 > 10 条或按领域可分时）

````markdown
### {分类标题}

#### {编号}. {规则标题}

**{规则陈述}**

- 适用场景：{什么情况下这条规则生效；可用表格表达判据}
- 原因：{为什么这样写 / 不这样写的错因}

> **注意**：{可选补充说明}
> **例外**：{可选补充说明}

**正确示例**（{文件路径}）：
```{lang}
{来自该仓库真实代码的短片段}
```

**错误示例**（{文件路径}，错误写法）[仅禁止规则必填]：
```{lang}
{该仓库中真实存在的反例片段}
```

**推荐替代写法** [仅禁止规则必填]：
```{lang}
{修正后的写法}
```
````

## 各字段要求

### 1. 规则标题

- 平铺结构：`### {编号}. {规则标题}`
- 分组结构：`#### {编号}. {规则标题}`，外覆 `### {分类标题}`
- 编号体系任选其一，同一语言规范文件内保持一致：
  - 纯数字：`1.`、`2.`、...，在章节内连续递增，禁止跳号、重复或无序
  - 分类编码：`ERR-GO-001`、`CON-GO-002`、...，同一文件内全局唯一，可跨文件引用；数字不要求连续，但编码前缀须反映领域分组
- 标题简洁，用名词短语概括规则核心
- 分类标题不编号，仅用于视觉/导航分组；分组内规则编号沿用章节全局编号，不因分组重置

### 2. 规则陈述

- 独占一行，加粗
- 必须以以下措辞开头：
  - 必须遵守的规则 → `**必须...**`
  - 推荐风格 → `**推荐...**`
  - 禁止反模式 → `**禁止...**`

### 3. 适用场景

- 用 bullet 列出规则生效的典型场景
- 必须给出"什么场景选哪个"的判据，不能只是枚举
- 允许使用 Markdown 表格表达多维度判据（如日志级别、并发原语选择矩阵），表格仅限本条规则内部

### 4. 原因

- 说明收益、错因或风险
- 禁止出现"因为外部规范要求""经团队讨论决定"等过程性表述

### 5. 注意与例外

- 可选字段，用于补充规则的边界条件或合法例外
- 必须具体说明例外场景和适用条件，不能空泛
- 如例外属于历史债务，优先写入 `## 历史例外说明` 章节

### 6. 正确示例

- 必须包含文件路径
- 代码片段必须来自该仓库真实代码
- 片段长度控制在 5–15 行以内，突出规则要点

### 7. 错误示例

- 仅禁止规则需要错误示例；必须遵守的规则和推荐风格规则不需要
- 必须标注"错误写法"
- 必须包含文件路径
- 必须给出错因说明

### 8. 推荐替代写法

- 仅禁止规则需要；必须遵守的规则和推荐风格规则不需要
- 必须给出可直接使用的替代写法，不能只描述原则

## 章节级规则清单

`coding-style-<lang>.md` 整体必须包含以下 5 个章节，顺序固定：

1. `## 适用范围`
2. `## 必须遵守的规则`
3. `## 推荐风格`
4. `## 禁止反模式`
5. `## 历史例外说明`

每个章节内，规则条目必须使用本模板格式。

## 历史例外说明

`## 历史例外说明` 章节中的每个例外必须包含：

1. 例外场景
2. 存在原因
3. 是否允许新代码继续使用
4. 建议迁移方式

## 禁止出现的内容

最终规范文件中禁止出现以下内容：

- 外部规范名称、引用、链接或版本号
- "与 XX 规范冲突""经讨论决定""参考 YY 规范第 N 条"等过程性描述
- 文件级或章节级的差异分析、gap 分析、对比表格
- formatter / linter 可自动修复的格式细节
- 构建 / 测试命令、覆盖率标准、lint 配置
- "来源：...""依据：..."等溯源标注

例外：单条规则条目内部允许使用 Markdown 表格表达判据（如日志级别、并发原语选择矩阵）。
