# devforge-test-design — 测试策略设计

## 概述

测试策略设计 Skill —— 测试分层定义 → 覆盖率目标分配 → 各级测试方案设计。

**核心原则**：
- **分层清晰**：单元测试、集成测试、性能测试的边界和职责明确划分
- **集成测试独立**：集成测试方案独立于特性 proposal，覆盖组件交互、多节点部署、故障恢复等全部跨进程场景
- **可反复迭代**：测试策略随架构和需求的迭代同步调整
- **领域适配**：根据 domain-config.yaml 的系统类型和质量属性优先级调整测试侧重点

## 何时使用

- 架构文档和需求文档初步就绪后，制定测试策略
- 架构或需求发生重大变更后，同步调整测试策略
- plan 阶段需要集成测试 proposal 的输入依据时

## 核心流程

### 1. 测试分层定义

**分层说明**：根据系统类型调整分层策略。

**通用分层**：

| 层级 | 职责 | 归属 |
|------|------|------|
| 单元测试 | 单函数/模块逻辑正确性（进程内、无外部依赖） | 包含在特性 proposal 中 |
| 集成测试 | 多节点部署、组件交互、接口契约、故障注入、端到端场景 | 独立 proposal |
| 性能测试 | 基准测试、压力测试、性能回归 | 独立 proposal |

**领域特定调整**（根据 domain-config.yaml 的 system.primary_type）：

- **分布式系统**：集成测试必须是多节点的，单机 mock 其他节点的"集成测试"无效。重点关注数据一致性验证、故障注入、网络分区。
- **数据库系统**：集成测试重点关注事务正确性、并发控制、查询优化器、故障恢复。
- **编译器/运行时**：集成测试重点关注端到端编译流程、优化 pass 正确性、运行时行为验证。
- **OS/内核**：集成测试重点关注系统调用正确性、进程调度、内存管理、文件系统。
- **网络协议栈**：集成测试重点关注协议正确性、拥塞控制、丢包恢复、性能基准。
- **嵌入式系统**：集成测试重点关注实时性保证、资源受限场景、硬件交互。

### 2. 覆盖率目标

**通用目标**：
- 单元测试：≥80% 行覆盖率（核心模块 ≥90%）
- 集成测试：覆盖所有关键路径、子系统交互接口、端到端场景、故障恢复场景
- 性能测试：建立基准线，回归检测 ≤5% 波动

**根据质量属性优先级调整**（读取 domain-config.yaml 的 quality_attributes.priorities）：
- 如果一致性优先级最高：集成测试重点验证数据一致性、并发正确性
- 如果性能优先级最高：性能测试覆盖更多场景，建立更细粒度的基准线
- 如果可用性优先级最高：集成测试重点验证故障恢复、降级策略

### 3. 各级测试方案

**集成测试方案**（根据系统类型调整）：

**通用验证项**：
- 组件交互验证：子系统间调用链路正确性
- 接口契约验证：接口输入输出符合 `docs/architecture/design.md` 中关键接口索引的定义
- 端到端场景：从客户端视角的完整流程

**领域特定验证项**（根据 domain-config.yaml 的 system.primary_type）：

**分布式系统**：
- 多节点部署验证：集群组网、成员变更、负载均衡
- 一致性协议验证：Raft/Paxos 在正常和异常场景下的正确性
- 故障注入：磁盘故障、网络分区、进程崩溃、慢节点、磁盘静默错误
- 故障恢复验证：节点宕机恢复、网络分区愈合、副本重建
- 数据一致性验证：副本一致性、写后读一致性、跨副本校验
- 升级回滚验证：在线升级、版本回滚、混合版本运行

**数据库系统**：
- 事务正确性：ACID 属性验证、隔离级别验证
- 并发控制：多事务并发执行、死锁检测与恢复
- 查询优化器：执行计划正确性、性能回归
- 故障恢复：崩溃恢复、日志回放、检查点

**编译器/运行时**：
- 端到端编译：源码 → 可执行文件的完整流程
- 优化 pass 正确性：各优化阶段的输出验证
- 运行时行为：内存管理、垃圾回收、异常处理
- 跨平台验证：不同目标平台的正确性

**OS/内核**：
- 系统调用正确性：各系统调用的输入输出验证
- 进程调度：调度策略正确性、公平性
- 内存管理：分配释放正确性、内存泄漏检测
- 文件系统：文件操作正确性、崩溃一致性

**网络协议栈**：
- 协议正确性：协议状态机验证、包格式验证
- 拥塞控制：拥塞算法正确性、公平性
- 丢包恢复：重传机制、乱序处理
- 性能基准：吞吐、延迟、CPU 使用率

**嵌入式系统**：
- 实时性保证：任务调度延迟、中断响应时间
- 资源受限场景：内存/CPU/功耗限制下的正确性
- 硬件交互：外设驱动正确性、DMA 操作

**性能测试方案**：
- 基准测试：IOPS、吞吐、延迟（P50/P99/P999）
- 压力测试：极端负载下的稳定性
- 性能回归检测：与基准线对比
