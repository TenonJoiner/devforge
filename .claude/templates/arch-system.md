### 故障假设

<明确定义系统在设计和测试时必须容忍的故障类型与范围>

| 故障类型 | 假设 | 系统级应对策略 |
|----------|------|---------------|
| Crash-stop | 任意进程随时崩溃，重启后数据可能丢失或损坏 | WAL + 定期 checkpoint；重启后从 WAL 恢复状态 |
| Network partition | 网络延迟、丢包、分区不可避免，无法区分节点崩溃与网络中断 | 基于心跳的超时检测 + 租约机制；分区时遵循预设的 CP/AP 策略 |
| Disk corruption | 磁盘可能因硬件故障产生位翻转或扇区损坏 | 端到端校验和（CRC32C / xxHash64）；定期 scrub 扫描 |
| Silent data corruption | 数据在传输或存储过程中可能静默损坏而不触发 IO 错误 | 每次网络传输和落盘必须携带校验和；读取时验证校验和 |
| Byzantine fault | `□ 假设存在 / □ 不假设存在`，若存在需说明阈值（如 f < N/3） | <若考虑拜占庭故障，说明对应的容错机制，如 BFT 变体> |
